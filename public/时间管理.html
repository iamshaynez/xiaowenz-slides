Act I · 03 / 20
我们先从「不是」开始

三个常见误区

大多数人理解的时间管理，一开始就错了。

不是「排满日程」
不是像甘特图一样，把时间切得井井有条地发配出去。那种「看起来很忙」的幻觉，恰恰是最隐蔽的陷阱。
不是「学习工具」
GTD、番茄钟、Notion……学习工具本身不是目的。工具只是容器，装不进去的注意力，再漂亮的界面也没用。
不是「只在工作中」
工作和生活共享了你作为一个人的时间和体能。时间管理一定是覆盖全部生活的一种习惯，而不是上班专属。

Dual Process

双流理论

创作和执行是两种完全不同的「心境」。

创作需要发散、联想、直觉；执行需要聚焦、逻辑、行动力。两种模式频繁切换，会导致巨大的效率损失——每次切换都是一次「认知税」。

MODE A
创作
发散 · 联想 · 不确定
MODE B
执行
聚焦 · 逻辑 · 可衡量
Peak Theory

峰值理论

情绪过高和过低，都会摧毁注意力。

峰值理论告诉我们：注意力不是线性资源，它随情绪波动。刻意安排正反馈，甚至适度的负反馈，才能控制整体心境，保持在高效区间。

情绪过高 → 过度兴奋 → 无法聚焦
情绪过低 → 动力枯竭 → 拖延逃避

Boundary

一个必要的假设

以上所有讨论，都建立在「体能精力恒定」的假设之上。

健身、更好的睡眠、规律的运动节奏——这些不是「时间管理」的范畴，但它们直接决定了你注意力池子的上限。

时间管理是「在既定容量下做最优分配」，而体能管理是「扩大容器本身」。两者缺一不可。
— 前提声明

Act II · 09 / 20
Habit

养成习惯，用起来

工具买了一堆，最后全靠脑子记——这是最普遍的失败模式。

每天打开它
不是「有空再看」，而是每天早上第一件事、睡前最后一件事。频率决定一切。
全部放进去
生活琐事、工作任务、灵感碎片——全部往里扔。不要侥幸「这个我记得住」，你记不住。
不要分类强迫症
先保证「全部在」再说整理。很多人花三小时建文件夹，最后一条任务没录。

Deconstruction

解构每一项任务

再复杂的目标，都应该被降级成可执行的「创作」或「执行」类 Task。

核心原则
按自己的节奏放到日程里，留点余量。任务Size尽可能小，维持能够持续关闭任务的正反馈。
MECE
mutually exclusive, collectively exhaustive —— 不重叠、不遗漏
MEASURABLE
是否完成是明确可衡量的，不要用「尽量」「争取」
SMALL
Size 尽可能小，小到「甜度」足够，让你想点掉它

Case Study · 减肥 10 斤

坏拆分 vs 好拆分

BAD · 坏拆分

每天跑 5 公里

  • 「每天跑5公里」和「减肥10斤」之间，没有必然逻辑关系
  • 「每天」是极高频承诺，一旦断档就是负反馈
  • 5公里对初学者「甜度」不足，难以持续
GOOD · 好拆分

减肥 = 摄入↓ + 消耗↑ + 测量

  • 降低摄入：每日控制摄入在目标值以下
  • 增加消耗：每日10分钟HIIT 或 30分钟慢跑
  • 定期测量：每月测体重体脂，修正日常规划

Case Study · 写 PPT

创意先行，执行跟上

写 PPT 的工作，天然包含两种心境。

PART A · 创意
理清脉络与故事线

这部分纯费脑，耗时极不确定。需要用「发散模式」——跑步、游泳、睡前、甚至带娃的时候都可以思考。不要强行安排在书桌前。

PART B · 执行
把内容组织成 PPT

有了「成片感」的骨架后，这部分费手但不费脑。依靠熟练度，耗时可控。安排在完整的专注时间窗口里一次性做完。

不要混在一起安排时间。创意用碎片时间和身体活动来孵化，执行用整块时间来推进。混在一起，两种心境互相干扰，结果两样都做不好。

Closing the Loop

事前想清，事后不漏

给自己留出两个固定时间。

PLAN
先想明白

事前想得越清楚，事情就越可控，浪费的心智就越少。Plan 不是浪费时间，是投资。每天花 10 分钟规划，省下的是整天的焦虑和返工。

REVIEW
不漏事儿

每天结束前 Review 一遍清单。不是检查「做了什么」，而是确认「还有什么没做」。漏掉的事情会在潜意识里持续消耗注意力。

还有一些「特殊」需求
提醒

结婚纪念、生日、喝水、信用卡还款、保险续费……这些不是任务，是防遗忘的哨兵。

跟踪

有趣新闻的后续、下属长周期任务、外部合作方依赖——不是自己做，但要定期看。

Back to Dual Process

双流理论返场

大部分工作都可以拆成「创作」和「执行」两类任务的组合。
如果我们自己都分不清，AI 大概率也做不好。

一个「复合型」任务直接丢给 AI，AI 的产出会四不像，人会更痛苦，并浪费最稀缺的「注意力」。

关键是：先拆分，再分别思考 AI 如何赋能。
— 核心原则

AI Integration

创作 + AI vs 执行 + AI

创作 + AI

提供灵感、选项、大纲

  • 不需要复杂的 Agent + Skill 工程环境
  • 任何 ChatBot 都是好选择
  • 目标是「线索信息」，不是最终成品
  • 标准思考逻辑、模板提示词 = 杠杆
工具：ChatGPT / Claude / 任何 LLM
执行 + AI

追求全自动化

  • 越少人工干预越好
  • 不需要通识智力，需要「输入→产出」能力
  • 核心模型 + Tools 的调用能力
  • 从重复劳动中彻底解放
工具：Coze / Qwen Studio / Claude Code + Skills
